Algorithm V0.1

Why Memory Hardness

The main reason why memory hardness is important is to make the proof of work function resistant to specialized hardware Memory hardness alleviates this problem by making the main limiting factor not CPU power, but memory. Thus, improvement through parallelization hits a roadblock: running ten memory-hard computations in parallel requires ten times as much memory.

Boolean DAG

Given a graph G composed of boolean operations with n nodes, it is connected by at most n-m binary boolean operations and at most n logical negation operations. The first m nodes are used as input nodes (the in-degree is 0) and the last k nodes are used as output nodes (the out-degree is 0). Now given the state of the next k nodes, find the state of m nodes that may cause this state.

Memory Consuming Calculation

This problem can meet the goal of memory size>memory bandwidth>computing power. Besides, after the boolean operation network is reasonably connected, the calculation of the last m nodes requires the participation of most of the previous nodes while the difficulty can be dynamically adjusted by the number of nodes and the number of connections. Additionally, the connection state can be related to the binary of the solution of the previous problem, which has the feasibility of forming a chain.
